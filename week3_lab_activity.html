▶ Paste into Colab — Cell 1: Setup
import numpy as np
import matplotlib.pyplot as plt
from sklearn.linear_model import Ridge, Lasso, ElasticNet, LogisticRegression
from sklearn.preprocessing import StandardScaler, PolynomialFeatures
from sklearn.pipeline import Pipeline
from sklearn.model_selection import train_test_split
from sklearn.datasets import make_regression, make_classification
np.random.seed(42)

# ── Dataset A: simple regression (for optimizer experiments) ────────────
N = 300
X_1d = 2 * np.random.rand(N, 1)
y_1d = 4 + 3 * X_1d.ravel() + np.random.randn(N) * 0.8
X_b  = np.c_[np.ones(N), X_1d]          # add bias column

# ── Dataset B: polynomial regression (for regularization experiments) ───
N2 = 80
X_poly_raw = np.sort(np.random.rand(N2) * 6 - 3)
y_poly = np.sin(X_poly_raw) + np.random.randn(N2) * 0.3
X_p, X_p_test = X_poly_raw[:60].reshape(-1,1), X_poly_raw[60:].reshape(-1,1)
y_p, y_p_test = y_poly[:60], y_poly[60:]

# ── Dataset C: many-feature regression (for Ridge/Lasso/sparsity) ───────
X_many, y_many = make_regression(n_samples=200, n_features=30,
                                  n_informative=5, noise=15, random_state=0)
X_many = StandardScaler().fit_transform(X_many)
X_m_tr, X_m_te, y_m_tr, y_m_te = train_test_split(X_many, y_many,
                                                    test_size=0.3, random_state=0)

# ── Dataset D: binary classification (for logistic regression) ──────────
X_clf, y_clf = make_classification(n_samples=300, n_features=2, n_redundant=0,
                                    n_informative=2, random_state=5, n_clusters_per_class=1)
X_clf = StandardScaler().fit_transform(X_clf)
X_c_tr, X_c_te, y_c_tr, y_c_te = train_test_split(X_clf, y_clf, test_size=0.3, random_state=0)

# ── Gradient descent helpers ──────────────────────────────────────────────
def gd(Xb, y, eta=0.1, n_epochs=80):
    theta = np.random.randn(Xb.shape[1])
    losses = []
    for _ in range(n_epochs):
        g = (2/len(y)) * Xb.T @ (Xb @ theta - y)
        theta -= eta * g
        losses.append(np.mean((Xb @ theta - y)**2))
    return theta, np.array(losses)

def momentum_gd(Xb, y, eta=0.1, beta=0.9, n_epochs=80):
    theta = np.random.randn(Xb.shape[1]); m = np.zeros_like(theta)
    losses = []
    for _ in range(n_epochs):
        g = (2/len(y)) * Xb.T @ (Xb @ theta - y)
        m = beta * m + (1 - beta) * g      # EMA of gradients
        theta -= eta * m
        losses.append(np.mean((Xb @ theta - y)**2))
    return theta, np.array(losses)

def adam_gd(Xb, y, eta=0.05, b1=0.9, b2=0.999, eps=1e-8, n_epochs=80):
    theta = np.random.randn(Xb.shape[1])
    m = np.zeros_like(theta); v = np.zeros_like(theta)
    losses = []
    for t in range(1, n_epochs + 1):
        g = (2/len(y)) * Xb.T @ (Xb @ theta - y)
        m = b1*m + (1-b1)*g; v = b2*v + (1-b2)*g**2
        mh = m/(1-b1**t); vh = v/(1-b2**t)
        theta -= eta * mh / (np.sqrt(vh) + eps)
        losses.append(np.mean((Xb @ theta - y)**2))
    return theta, np.array(losses)

print("Setup complete. Datasets A–D ready.")

Part 1 — Optimization Teams (Teams 1–4)

Loss surfaces, learning rates, momentum, and Adam — using Dataset A.

Learning Rate

Team 1 — The Learning Rate Disaster Tour

Mission: Run GD with 4 different learning rates and plot all 4 loss curves on one figure.

Code hint: Call gd(X_b, y_1d, eta=???) four times.

Try: eta = 0.001, 0.05, 0.2, 0.6

Observe: Which converges fastest? Which diverges or oscillates? Which barely moves?

Explain: What does the loss curve shape tell you about whether eta is too big, too small, or just right?

Expected insight: Small eta → slow but stable. Large eta → oscillation or divergence (loss goes up). There's a "Goldilocks zone" — and finding it is half the job of hyperparameter tuning.
Loss Surface

x1 = np.random.rand(200) * 1000   # large scale

Momentum

Mission: On Dataset A, compare plain GD vs Momentum at the same learning rate. Plot loss curves side by side.

Code hint: Call both gd() and momentum_gd() with eta=0.05. Add gradient noise to GD to see the smoothing effect:

np.random.seed(7)

Adam

Team 4 — GD vs Momentum vs Adam Race

Mission: Run all three optimizers on Dataset A and plot their loss curves together. Then try different learning rates for each to find their "best" eta.

Code hint: Call gd(), momentum_gd(), and adam_gd(). Use n_epochs=100.

Try: SGD at eta=0.1, Momentum at eta=0.05, Adam at eta=0.05

Observe: Which reaches a low loss fastest? How does each react to a 2× larger learning rate?

Expected insight: Adam typically converges faster and is more robust to learning rate choice. Its advantage grows on complex (non-convex) problems. On simple linear regression they all converge to the same minimum eventually.

Part 2 — Regularization Teams (Teams 5–7)

Overfitting, Ridge, Lasso, Elastic Net — using Dataset B (polynomial) and Dataset C (sparse).

Ridge

Team 5 — Overfitting vs Ridge

Mission: Fit degree-12 polynomial regression on Dataset B with and without Ridge regularization. Show how regularization prevents overfitting.

Code hint:

from sklearn.linear_model import Ridge
from sklearn.preprocessing import PolynomialFeatures
from sklearn.pipeline import Pipeline

X_plot = np.linspace(-3, 3, 300).reshape(-1,1)
for alpha, color, label in [(0, 'tomato', 'No reg (overfit)'),
                              (0.1, 'orange', 'Ridge α=0.1'),
                              (10, 'steelblue', 'Ridge α=10')]:
    model = Pipeline([('poly', PolynomialFeatures(12)),
                      ('scl', StandardScaler()),
                      ('reg', Ridge(alpha=alpha))])
    model.fit(X_p, y_p)
    plt.plot(X_plot, model.predict(X_plot), color=color, label=label)
plt.scatter(X_p, y_p, s=18, alpha=0.6, color='gray')
plt.ylim(-2, 2); plt.legend(); plt.title('Ridge Regularization')
plt.show()

Observe: How does increasing α change the smoothness of the fit? What does α=0 look like?

Expected insight: α=0 wildly overfits the training data. Higher α forces the model toward simpler, smoother predictions. The sweet spot generalizes to new data.
Lasso

Team 6 — Lasso's Sparsity Superpower

Mission: Using Dataset C (30 features, only 5 informative), compare Ridge and Lasso coefficient patterns. Show that Lasso zeros out irrelevant features.

Code hint:

ridge = Ridge(alpha=1.0).fit(X_m_tr, y_m_tr)
lasso = Lasso(alpha=0.5).fit(X_m_tr, y_m_tr)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))
for ax, model, name in [(axes[0], ridge, 'Ridge'), (axes[1], lasso, 'Lasso')]:
    ax.bar(range(30), model.coef_, color='steelblue', alpha=0.7)
    ax.axhline(0, color='black', lw=0.8)
    n_zero = np.sum(np.abs(model.coef_) < 0.01)
    ax.set_title(f'{name} — {n_zero}/30 coefficients ≈ 0')
    ax.set_xlabel('Feature index')
plt.tight_layout(); plt.show()

Observe: How many features does Lasso zero out? Does Ridge zero anything?

Try: Increase Lasso α from 0.1 → 1.0 → 5.0. What happens?

Expected insight: Ridge shrinks all coefficients but keeps all non-zero. Lasso performs automatic feature selection — zeroing out irrelevant features entirely. At very high α, Lasso zeros everything (underfitting).
Elastic Net

Team 7 — Elastic Net: Best of Both Worlds

Mission: On Dataset C, compare Ridge (l1_ratio=0), Elastic Net (l1_ratio=0.5), and Lasso (l1_ratio=1) coefficient patterns and test MSE.

Code hint:

from sklearn.linear_model import ElasticNet

fig, axes = plt.subplots(1, 3, figsize=(14, 4))
for ax, l1, name in [(axes[0], 0.0, 'Ridge (l1=0)'),
                      (axes[1], 0.5, 'Elastic Net (l1=0.5)'),
                      (axes[2], 1.0, 'Lasso (l1=1)')]:
    en = ElasticNet(alpha=0.5, l1_ratio=l1, max_iter=5000).fit(X_m_tr, y_m_tr)
    test_mse = np.mean((en.predict(X_m_te) - y_m_te)**2)
    n_zero = np.sum(np.abs(en.coef_) < 0.01)
    ax.bar(range(30), en.coef_, color='steelblue', alpha=0.7)
    ax.axhline(0, color='black', lw=0.8)
    ax.set_title(f'{name}\nMSE={test_mse:.1f}  zeros={n_zero}/30')
plt.tight_layout(); plt.show()

Observe: Which achieves the best test MSE? Does the l1_ratio control the sparsity?

Expected insight: Elastic Net captures correlated features better than pure Lasso (which arbitrarily picks one from a group). It offers sparsity like Lasso AND stability like Ridge. l1_ratio is the dial between the two behaviors.

Part 3 — Early Stopping & Classification (Teams 8–10)

When to stop training, and moving from regression to probability estimation.

Early Stopping

Team 8 — Catching the Overfitting Moment

Mission: Train a high-degree polynomial model incrementally and plot both training and validation loss. Find the epoch where validation loss starts rising.

Code hint:

from sklearn.linear_model import SGDRegressor
from sklearn.preprocessing import PolynomialFeatures, StandardScaler
from sklearn.pipeline import Pipeline

poly_pipe = Pipeline([('poly', PolynomialFeatures(8)), ('scl', StandardScaler())])
X_tr_f = poly_pipe.fit_transform(X_p)
X_te_f = poly_pipe.transform(X_p_test)

sgd = SGDRegressor(max_iter=1, warm_start=True, learning_rate='constant',
                   eta0=0.0005, random_state=42)
train_l, val_l, best = [], [], (np.inf, 0)
for epoch in range(300):
    sgd.fit(X_tr_f, y_p)
    tr  = np.mean((sgd.predict(X_tr_f)  - y_p)**2)
    val = np.mean((sgd.predict(X_te_f) - y_p_test)**2)
    train_l.append(tr); val_l.append(val)
    if val < best[0]: best = (val, epoch)

plt.figure(figsize=(10, 4))
plt.plot(train_l, label='Train', color='steelblue', lw=2)
plt.plot(val_l,   label='Validation', color='tomato', lw=2)
plt.axvline(best[1], color='green', ls='--', lw=1.5, label=f'Best epoch={best[1]}')
plt.yscale('log'); plt.xlabel('Epoch'); plt.ylabel('MSE (log)')
plt.title('Early Stopping: Find the Overfitting Point')
plt.legend(); plt.tight_layout(); plt.show()
print(f'Best validation epoch: {best[1]}, val MSE: {best[0]:.4f}')

Observe: At what epoch do train and val loss diverge? What does that gap mean?

Expected insight: The gap between train and val loss is the generalization gap. The best model is saved at the minimum of the val curve — training past this point memorizes training noise and hurts generalization.
Logistic Regression

Team 9 — Decision Boundary & Regularization C

Mission: Using Dataset D, train logistic regression with 3 different C values and visualize how the decision boundary changes.

Code hint:

xx, yy = np.meshgrid(np.linspace(-3,3,200), np.linspace(-3,3,200))
grid = np.c_[xx.ravel(), yy.ravel()]

fig, axes = plt.subplots(1, 3, figsize=(14, 4))
for ax, C, color in [(axes[0], 0.01, 'Strong reg'),
                      (axes[1], 1.0,  'Default'),
                      (axes[2], 100,  'Weak reg')]:
    clf = LogisticRegression(C=C, random_state=0).fit(X_c_tr, y_c_tr)
    Z = clf.predict(grid).reshape(xx.shape)
    acc = clf.score(X_c_te, y_c_te)
    ax.contourf(xx, yy, Z, alpha=0.25, cmap='RdYlBu')
    ax.scatter(X_c_te[:,0], X_c_te[:,1], c=y_c_te, cmap='RdYlBu', s=20, edgecolors='k', lw=0.5)
    ax.set_title(f'C={C}  |  Test Acc={acc:.2f}')
plt.tight_layout(); plt.show()

Observe: Small C = strong regularization. Does the boundary become more linear? Does test accuracy change?

Expected insight: C=1/λ in logistic regression — small C means high regularization → simpler (more linear) boundary. Very small C underfits. Very large C overfits to training points. Sweet spot generalizes best.
Probability

Team 10 — Reading the Sigmoid Output

Mission: Train logistic regression on Dataset D, then explore the predicted probabilities. Show the sigmoid curve and a calibration plot.

Code hint:

clf = LogisticRegression(C=1.0, random_state=0).fit(X_c_tr, y_c_tr)
probs = clf.predict_proba(X_c_te)[:, 1]    # P(class=1)

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

# Sigmoid shape
z = np.linspace(-5, 5, 200)
axes[0].plot(z, 1/(1+np.exp(-z)), color='steelblue', lw=2.5)
axes[0].axhline(0.5, color='gray', ls='--', lw=1)
axes[0].set_title('Sigmoid: score → probability')
axes[0].set_xlabel('Linear score (w·x + b)'); axes[0].set_ylabel('P(y=1)')

# Predicted probability histogram
axes[1].hist(probs[y_c_te==0], bins=20, alpha=0.6, color='tomato',    label='True class 0')
axes[1].hist(probs[y_c_te==1], bins=20, alpha=0.6, color='steelblue', label='True class 1')
axes[1].axvline(0.5, color='black', ls='--', lw=1.5, label='Decision threshold 0.5')
axes[1].set_title('Predicted P(y=1) by True Class')
axes[1].set_xlabel('Predicted probability'); axes[1].legend()

plt.tight_layout(); plt.show()
print(f'Confident wrong (prob>0.9, wrong class): {np.sum((probs>0.9) & (y_c_te==0))}')

Observe: Are the two histograms well-separated? How many examples is the model confidently wrong about?

Expected insight: Well-separated histograms = the model is confident and correct. Overlap near 0.5 = uncertain cases. The decision threshold (default 0.5) can be moved to trade off precision vs recall.

Instructor Wrap-Up: The Week 3 Mental Model

Tie all 10 team insights into one coherent story.

Teams 1–2 taught us
  • Learning rate is the most sensitive hyperparameter
  • Loss surface shape = feature scales
  • Always scale → circular surface → GD works
Teams 3–4 taught us
  • Momentum filters gradient noise via EMA
  • Adam = Momentum + RMSProp, best default
  • All optimizers find same minimum; speed differs
Teams 5–7 taught us
  • Regularization = penalty on coefficient size
  • Ridge: shrink all. Lasso: zero out irrelevant.
  • Elastic Net: blend both for correlated features
Teams 8–10 taught us
  • Training loss always falls; val loss has a minimum
  • Early stopping = save best val checkpoint
  • Logistic: C controls boundary complexity
  • Sigmoid converts scores to calibrated probabilities
The one-sentence summary: Every technique in Week 3 is a targeted solution to one specific failure mode — scaling fixes loss surface geometry, momentum filters gradient noise, regularization prevents coefficient explosion, early stopping catches the generalization peak, and logistic regression extends all of this to classification.
